# Internationalization (5 languages)

## Mobile (Expo)

- **Libraries:** `i18next`, `react-i18next`, `expo-localization`, `@react-native-async-storage/async-storage`
- **Locales:** `en`, `ar`, `fr`, `es`, `hi` — JSON files in `src/i18n/locales/`
- **Persistence:** chosen language stored under `alloul_language` in AsyncStorage
- **RTL:** Arabic (`ar`) triggers `I18nManager.forceRTL`; a one-time app restart may be needed on some Android builds for perfect layout
- **API:** `src/lib/api.ts` sends header **`Accept-Language: <current i18n.language>`** on every request for future backend use

## Changing copy

1. Edit the key in all five `*.json` files under `src/i18n/locales/` (keep keys identical across files).
2. Use `useTranslation()` and `t('namespace.key')` in components.

## Settings UI

Users pick language in **Settings** (tab **Menu** in workspace, or via Drawer → Settings).

## Backend

There is no requirement that the FastAPI app returns translated strings yet; the header is forwarded so you can add catalogs or DB-driven translations when ready.
